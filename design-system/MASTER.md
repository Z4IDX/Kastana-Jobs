# Design System — Kastana Jobs (MASTER)

> Global source of truth for UI/UX decisions. Generated by `uipro init` (ui-ux-pro-max skill).
> Page-specific deviations live in `design-system/pages/<page>.md` and **override** this file.
> When building a page, read this file first, then check for a matching page override.

## Product profile
- **Type**: Curated job board (content-first directory + light SaaS employer/admin dashboards).
- **Audience**: Arabic-first C-end job seekers (browse, no account) + employers (self-serve postings) + platform admins.
- **Stack**: Plain PHP 8 + MySQL on XAMPP. Vanilla CSS (custom properties) + a little vanilla JS. **No build step, no framework, no bundler.**
- **Direction**: Bilingual, **RTL by default (Arabic)** with an English (LTR) toggle. Every visual rule below must hold in both directions.
- **Tone**: Trustworthy, clean, professional. Content and legibility over decoration.

## Palette (matches `:root` tokens in `assets/css/style.css` — do not diverge)
> Historical variable names are kept intentionally: `--chestnut`/`--honey` now hold blue/yellow.

| Token | Value | Role |
|-------|-------|------|
| `--chestnut` | `#1D5C9D` | Brand blue — primary buttons, links, hero, card spine |
| `--chestnut-deep` | `#164A80` | Darker blue — hero / admin-bar backgrounds |
| `--honey` | `#EAA62C` | Brand yellow — accents, highlights, badges |
| `--ink` | `#57575A` | Brand grey — body text |
| `--brand-orange` | `#F26E2C` | Secondary accent |
| `--brand-green` | `#5A9E43` | Success / positive |
| Admin status | green=approved · amber=pending · red/orange=rejected | Status semantics |

**Rules**: use tokens, never raw hex in components (`color-semantic`). Functional color (approved/pending/rejected, error/success) must also carry an icon or text label, never color alone (`color-not-only`, `color-not-decorative-only`). Foreground/background pairs must meet **4.5:1** (`color-accessible-pairs`) — verify grey `--ink` on light surfaces especially.

## Typography (matches loaded fonts)
- **Display**: Fraunces. **Body**: Plus Jakarta Sans. **Labels/mono**: Space Mono.
- **RTL**: Cairo + Tajawal load when Arabic is active — keep the Arabic pairing in sync when changing headings.
- Body ≥ **16px** (`readable-font-size` — avoids iOS auto-zoom). Line-height **1.5–1.75** (`line-height`).
- Line length: mobile 35–60 chars, desktop 60–75 (`line-length-control`).
- Weight hierarchy: headings 600–700, labels 500, body 400 (`weight-hierarchy`).
- Use tabular figures for salary ranges / counts so rows don't shift (`number-tabular`).

## Layout & spacing
- **Mobile-first**, scale up. Breakpoints 375 / 768 / 1024 / 1440 (`mobile-first`, `breakpoint-consistency`).
- 4/8px spacing rhythm (`spacing-scale`). Consistent desktop container max-width (`container-width`).
- No horizontal scroll on mobile (`horizontal-scroll`). Never disable zoom; keep `width=device-width, initial-scale=1` (`viewport-meta`).
- Prefer `min-h-dvh` over `100vh` on mobile (`viewport-units`).
- Hierarchy via size/spacing/contrast, not color alone (`visual-hierarchy`).

## RTL discipline (project-critical)
- Everything must mirror correctly when `dir="rtl"`: use logical properties (`margin-inline`, `padding-inline`, `inset-inline`, `text-align: start/end`) over left/right.
- Icons that imply direction (arrows, "next/prev", back) must flip in RTL.
- Test every new component in **both** Arabic (RTL) and English (LTR) before delivery.

## Interaction & accessibility (CRITICAL — never ship without)
- Contrast 4.5:1 normal text / 3:1 large (`color-contrast`). Visible focus rings, never removed (`focus-states`).
- Icon-only buttons need `aria-label` (`aria-labels`). Every input has a real `<label>` (`form-labels`, not placeholder-only).
- Touch targets ≥ 44×44px with ≥8px spacing (`touch-target-size`, `touch-spacing`).
- `cursor: pointer` on clickable elements (`cursor-pointer`). Primary actions on tap/click, not hover-only (`hover-vs-tap`).
- Buttons show loading + disable during async (`loading-buttons`). Errors appear next to their field (`error-placement`).
- Respect `prefers-reduced-motion` (`reduced-motion`). Animations 150–300ms, transform/opacity only (`duration-timing`, `transform-performance`).
- No emoji as structural icons — use SVG (Heroicons / Lucide) (`no-emoji-icons`).

## Forms (register / login / post-job / admin edit)
- Visible labels, required markers, helper text below complex inputs (`input-labels`, `required-indicators`, `input-helper-text`).
- Validate on blur, not per keystroke; error states state cause + fix (`inline-validation`, `error-clarity`).
- Semantic input types (`email`, `tel`, `number`) for correct mobile keyboards (`input-type-keyboard`).
- Submit → loading → success/error; confirm before destructive actions (approve/reject/delete) (`submit-feedback`, `confirmation-dialogs`).
- After a submit error, focus the first invalid field (`focus-management`). Mobile input height ≥44px (`touch-friendly-input`).
- These map to existing server-side validation in `employer/post-job.php` and `admin/edit-job.php` — keep client affordances consistent with those rules.

## Job board specifics
- **Job cards** (`includes/job-card.php`): clear title → company → location/type → salary hierarchy; featured + expired badges must be readable in both themes and carry text, not just color. Save-star needs an `aria-label` and ≥44px hit area.
- **Board** (`index.php`): search/filter/sort controls reachable and labeled; show a helpful empty state when no jobs match (`empty-states`), not a blank list.
- **Job detail** (`job.php`): apply-by Call (`tel:`) / Email (`mailto:`) / apply_url are the primary CTAs — one clear primary action, others subordinate (`primary-action`).
- **Dashboards** (employer/admin): denser spacing is fine; keep status colors + labels; approval popups must not steal focus (`toast-accessibility`, `aria-live` for announcements).

## Anti-patterns to avoid
- Raw hex in components; color-only status meaning; placeholder-as-label.
- Left/right CSS that breaks RTL; un-flipped directional icons.
- Emoji icons; removed focus outlines; hover-only actions; sub-44px tap targets.
- Animating width/height/top/left; motion without `prefers-reduced-motion` fallback.
- Adding a build step / framework / CDN beyond the existing Google Fonts (CSP is same-origin + Google Fonts only).

## Pre-delivery checklist
- [ ] Renders correctly in Arabic (RTL) **and** English (LTR).
- [ ] Uses `--*` tokens, `e()`/`t()`/`th()` escaping conventions, `csrf_field()` on forms.
- [ ] Contrast ≥4.5:1; visible focus; labels on inputs & icon buttons.
- [ ] Touch targets ≥44px; tested at 375px and landscape.
- [ ] Status conveyed by text+icon, not color alone.
- [ ] No new build tooling; runnable by copying to `htdocs`.
